---
title: Server-Sent Events
---

The dev server exposes Server-Sent Events (SSE) endpoints for real-time bundler state and app client logs. This is useful for integrating with external tools such as LLMs, CI dashboards, or custom monitoring.

## SSE Endpoints

```
GET /sse/events
GET /sse/client-logs
```

`/sse/events` streams build, device, server, and cache events. `/sse/client-logs` streams app console logs forwarded by the HMR client.

Each endpoint returns a `text/event-stream` response. Each event is formatted as:

```
event: <event_type>
data: <json_payload>
```

Bundler-scoped events include `bundlerId`. Use this value with the MCP `get_bundler_status` tool when polling the latest state for a specific bundle. Device, server, and cache events are not tied to a single bundler instance. Client log events are streamed separately from build events.

### `/sse/events` Event Types

#### `bundle_build_started`

Emitted when a bundle build starts.

```json
{ "type": "bundle_build_started", "bundlerId": "<bundlerId>" }
```

#### `bundle_build_done`

Emitted when a bundle build completes successfully.

```json
{
  "type": "bundle_build_done",
  "bundlerId": "<bundlerId>",
  "totalModules": 1270,
  "transformedModules": 1200,
  "cacheHitModules": 70,
  "duration": 1523.45,
  "bundleFilePath": "/path/to/project/.rollipop/bundles/<bundlerId>.bundle"
}
```

`bundleFilePath` points to the file-system copy of the generated bundle. Rollipop keeps the bundle in memory and also writes it to this path; when the file is edited after the build, Rollipop serves the edited file content instead of the in-memory copy. This is intended for bundle-level debugging by tools and agents.

#### `bundle_build_failed`

Emitted when a bundle build fails.

```json
{
  "type": "bundle_build_failed",
  "bundlerId": "<bundlerId>",
  "error": "SyntaxError: Unexpected token"
}
```

#### `watch_change`

Emitted when a file change is detected.

```json
{ "type": "watch_change", "bundlerId": "<bundlerId>", "file": "src/App.tsx" }
```

#### `device_connected` / `device_disconnected`

Emitted when a device connects or disconnects via HMR.

```json
{ "type": "device_connected", "clientId": 0 }
```

#### `server_ready`

Emitted once when the dev server is ready.

```json
{ "type": "server_ready", "host": "localhost", "port": 8081 }
```

#### `cache_reset`

Emitted when the MCP `reset_cache` tool clears the build cache.

```json
{ "type": "cache_reset" }
```

### `/sse/client-logs` Event Types

#### `client_log`

Emitted when the app logs to the console through the HMR client.

```json
{ "type": "client_log", "bundlerId": "<bundlerId>", "data": ["Hello from the app"] }
```

`bundlerId` is included when the HMR client has already been associated with a bundler instance. It may be omitted for logs received before that association is available.

## Usage Examples

### curl

```bash
# Stream build and dev-server events
curl -N http://localhost:8081/sse/events

# Stream app client logs
curl -N http://localhost:8081/sse/client-logs
```

### Node.js

```ts
import http from 'node:http';

const req = http.get(
  'http://localhost:8081/sse/events',
  { headers: { Accept: 'text/event-stream' } },
  (res) => {
    res.setEncoding('utf8');
    res.on('data', (chunk) => {
      for (const message of chunk.split('\n\n')) {
        if (!message.trim() || message.startsWith(':')) continue;

        let event = '';
        let data = '';
        for (const line of message.split('\n')) {
          if (line.startsWith('event: ')) event = line.slice(7);
          if (line.startsWith('data: ')) data = line.slice(6);
        }

        if (event && data) {
          console.log(event, JSON.parse(data));
        }
      }
    });
  },
);
```
