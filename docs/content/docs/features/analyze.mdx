---
title: Bundle Analysis
---

Rollipop can generate an interactive bundle analysis report, helping you identify large modules and optimize bundle size.

## Usage

```ts title="rollipop.config.ts"
import { defineConfig } from 'rollipop';

export default defineConfig({
  analyzer: {
    enabled: true,
  },
});
```

When enabled, Rollipop generates two outputs:

- **JSON data file**: Raw analysis data of the bundle
- **HTML report**: Interactive visualization generated from the data

In build mode, files are written to the bundle output directory:

- `analyze-data.json`
- `report.html`

In development mode, there is no bundle output directory, so Rollipop writes id-scoped files under the project shared data directory:

- `.rollipop/analyze/<bundler-id>.json`
- `.rollipop/analyze/<bundler-id>.html`

You can also upload the JSON data file to the [Analyzer](https://leegeunhyeok.github.io/rolldown-analyzer) web app.

## Options

### enabled

- **Type:** `boolean`
- **Default:** `false`

Enable bundle analysis.

### analyzeFile

- **Type:** `string`
- **Default:** `'analyze-data.json'`

Output filename for the analysis data in build mode.

### reportFile

- **Type:** `string`
- **Default:** `'report.html'`

Output filename for the generated HTML report in build mode.

### autoOpen

- **Type:** `boolean`
- **Default:** `false`

Automatically open the generated report in the browser after a build completes.

```ts
export default defineConfig({
  analyzer: {
    enabled: true,
    autoOpen: true,
  },
});
```
