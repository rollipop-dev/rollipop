title: Troubleshooting

This guide helps you diagnose and resolve common issues with Rollipop.

## Enable Debug Logging

Rollipop provides detailed debug and trace logs to help diagnose issues. These logs are disabled by default for performance reasons.

### Enable Debug Mode

Set the `DEBUG_ROLLIPOP` environment variable to enable debug and trace logging:

```bash
# Enable debug logging
DEBUG_ROLLIPOP=1 rollipop start

# Equivalent
DEBUG_ROLLIPOP=true rollipop start

# Or for bundle command
DEBUG_ROLLIPOP=1 rollipop bundle --entry-file ./index.js --bundle-output ./dist/main.bundle.js
```

When debug mode is enabled, Rollipop prints additional debug logs for the running command or dev server process.

## Programmatic Usage

When using Rollipop programmatically, you can also enable debug logging:

```ts
// Set the environment variable before importing
process.env.DEBUG_ROLLIPOP = '1';

import * as Rollipop from 'rollipop';

const config = await Rollipop.loadConfig();
const server = await Rollipop.runServer(config, { port: 8081 });
```

## Inspect Generated Bundles

After a dev server build completes, Rollipop writes the generated bundle to:

```txt
.rollipop/bundles/<id>.bundle
```

You can inspect this file to debug the generated bundle code. If you edit the file after the build, the dev server serves the edited bundle content instead of the in-memory copy. This is intended for bundle-level debugging and for quickly testing a bundle edit before moving the real fix back to source, config, or plugin code.

## Reporting Issues

If you encounter an issue that isn't covered in this guide:

1. **Search existing issues**: Check if someone has already reported the issue at [github.com/rollipop-dev/rollipop/issues](https://github.com/rollipop-dev/rollipop/issues)

2. **Gather information**:

- Rollipop version (`rollipop --version`)
- Node.js version (`node --version`)
- Operating system
- Debug logs (run with `DEBUG_ROLLIPOP=1`)
- Minimal reproduction steps

3. **Create a new issue**: If the issue hasn't been reported, create a new issue at [github.com/rollipop-dev/rollipop/issues/new](https://github.com/rollipop-dev/rollipop/issues/new)
